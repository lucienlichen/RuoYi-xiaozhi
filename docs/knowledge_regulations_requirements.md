# 前沿知识AI助手 & 法规标准AI助手 功能需求文档

> **版本：** v2.0  
> **所属项目：** 起重装备全生命周期数据AI智能体平台（EMDS-V2）  
> **更新日期：** 2026-03-31  
> **版本说明：** v2.0 将原模块九（前沿知识AI助手）和模块十（法规标准AI助手）合并重写，聚焦第一阶段核心交付：文档列表浏览与内容查看，RAG问答作为规划功能纳入。

---

## 一、模块概述

**模块名称：** 知识与法规服务模块

**模块定位：** 整合前沿知识AI助手和法规标准AI助手两个子模块，第一阶段实现文档结构化列表展示与内容详情查看，前端适配普通PC与8寸竖屏两种终端。

### 1.1 子模块总览

| 子模块 | 名称 | 颜色标识 | 路由 | 定位 |
|--------|------|---------|------|------|
| safety_maintenance_ai | 前沿知识AI助手 | Rose（玫红色） | `/app?service=safety_maintenance_ai` | 将安全维保领域的核心电子书按章节结构化展示，用户可按章节标题浏览并查看图文内容 |
| regulations_ai | 法规标准AI助手 | Slate（石板灰） | `/app?service=regulations_ai` | 内置起重机相关法律法规与技术规范文档，按分类列表展示法规标题，用户点击后查看法规全文内容 |

---

## 二、响应式设计规范

两个子模块均需适配以下两种终端屏幕：

### 2.1 普通PC端

| 属性 | 规格 |
|------|------|
| 最小宽度 | ≥ 1024px |
| 布局方式 | 左右分栏布局，左侧列表面板固定宽度，右侧内容查看区自适应剩余宽度 |
| 左侧面板宽度 | 320px |
| 右侧内容区 | 自适应剩余宽度 |
| 详情展示方式 | 右侧内联弹窗（抽屉/侧滑面板），宽度占主内容区 60%-70% |

### 2.2 8寸竖屏端

| 属性 | 规格 |
|------|------|
| 参考分辨率 | 约 800×1280px（竖屏） |
| 布局方式 | 单栏全屏切换布局，列表页与详情页通过导航栈切换，非同屏并列 |
| 列表视图 | 全屏展示列表，点击项目后推入详情页 |
| 详情视图 | 全屏展示内容，顶部提供返回按钮回到列表 |
| 导航模式 | 列表 → 详情 单层导航栈，支持手势返回 |

### 2.3 通用适配规则

| 规则 | 说明 |
|------|------|
| 字体大小 | 8寸竖屏下适当放大，正文不小于 16px，确保可读性 |
| 点击热区 | 8寸竖屏下不小于 44×44px，适配触控操作 |
| 列表间距 | 8寸竖屏下适当增大，防止误触 |
| 图片交互 | 详情页内图片自适应宽度，8寸竖屏端支持手势缩放 |

## 三、前沿知识AI助手（第一阶段）

**模块ID：** `safety_maintenance_ai`

**功能定位：** 将两本核心安全维保电子书按章节进行结构化处理，以章节目录树形式展示，点击章节标题可查看该章节的完整文字内容及插图。

### 3.1 数据来源与结构

**数据类型：** 电子书（2本核心书籍）

**预处理方式：** 管理端预处理，将电子书按章节拆分为结构化数据

**数据结构定义：**

```
Book（书籍）
├── id          // 唯一标识
├── title       // 书名
├── author      // 作者
├── cover_image // 封面图片URL（可选）
└── chapters[]  // 章节数组
    ├── id           // 章节唯一标识
    ├── title        // 章节标题
    ├── order        // 排序序号
    ├── level        // 层级（1=章, 2=节, 3=小节）
    ├── content_html // 章节正文内容（富文本HTML，含文字与插图）
    ├── images[]     // 插图数组
    │   ├── id       // 图片标识
    │   ├── url      // 图片资源URL
    │   ├── caption  // 图片说明文字
    │   └── position // 在正文中的插入位置标记
    └── children[]   // 子章节数组（递归结构）
```

### 3.3 功能需求清单

| 编号 | 功能名称 | 优先级 | 需求描述 |

## 四、法规标准AI助手（第一阶段）

**模块ID：** `regulations_ai`

**功能定位：** 展示起重机相关法律法规文档列表，按分类标签组织，用户点击法规标题后查看该法规的完整内容。法规文档由管理端上传 PDF/Word 等格式文件，业务端仅做列表展示与内容查看。

### 4.1 数据来源

#### 管理端（上传与管理）

| 属性 | 说明 |
|------|------|
| 支持格式 | PDF（.pdf）、Word（.doc / .docx）、纯文本（.txt） |
| 上传流程 | 管理端上传文件 → 后端解析存储 → 业务端自动可见 |
| 元数据字段 | 文档标题、所属分类、发布日期（可选）、文号（可选） |

#### 业务端（仅查看）

业务端不提供上传功能，仅支持浏览分类列表和查看文档全文内容。

### 4.2 法规分类

| 分类ID | 分类名称 | 说明 |

| tsg | TSG技术规范 | 特种设备安全技术规范 |

## 五、规划功能（后续版本）

> 以下功能不在第一阶段交付范围内，将在后续版本中实现。

### 5.1 RAG智能问答 — 前沿知识

**编号：** P-01  
**所属模块：** 前沿知识AI助手（safety_maintenance_ai）

基于两本电子书的结构化内容构建向量知识库，用户可在对话框中用自然语言提问，AI基于书籍内容生成回答并标注章节来源与页码。

**规划能力：**
- 自然语言问答对话界面
- 答案引用来源章节与页码标注
- 多轮对话上下文记忆
- 对话历史本地持久化与清空

### 5.2 RAG智能问答 — 法规标准

**编号：** P-02  
**所属模块：** 法规标准AI助手（regulations_ai）

基于内置及用户上传的法规文档构建向量知识库，用户可提问法规相关问题，AI精准引用具体法规条文作答。

**规划能力：**
- 基于法规内容的专业问答
- 引用具体法规条文编号与原文
- 跨多部法规综合分析
- 法规适用性判断建议

### 5.3 用户自主上传知识文档

**编号：** P-03  
**所属模块：** 前沿知识AI助手（safety_maintenance_ai）

允许用户在前沿知识助手中自行上传 PDF/Word/TXT/Markdown 等文档扩充知识库，上传后自动解析入库。

**规划能力：**
- 多格式文档上传（PDF/Word/TXT/Markdown）
- 文档解析状态追踪（待解析 → 解析中 → 已就绪）
- 解析失败重试机制

### 5.4 全文搜索

**编号：** P-04  
**所属模块：** 前沿知识AI助手 & 法规标准AI助手（两者共用）

在知识文档和法规文档中提供关键词全文检索功能，搜索结果高亮并定位到具体章节或条文。

**规划能力：**
- 关键词搜索输入框
- 搜索结果列表（含上下文摘要）
- 点击搜索结果跳转并高亮定位

## 七、技术说明

| 项目 | 说明 |
|------|------|
| 技术栈 | Vue 3 + TypeScript + Pinia + Tailwind CSS + Vite（与主项目保持一致） |
| 响应式方案 | 使用 Tailwind CSS 响应式类 + CSS 媒体查询实现双端适配，8寸竖屏以 `max-width: 820px` 为断点 |
| 内容渲染 | 章节正文使用富文本 HTML 渲染，插图采用懒加载策略 |
| 数据持久化 | 第一阶段前端使用 Pinia 状态管理 + 静态 JSON 数据，后续对接后端 API |
| 图片处理 | 插图资源统一存储于后端静态资源服务，前端通过 URL 引用加载 |

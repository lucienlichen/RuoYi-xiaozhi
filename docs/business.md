# 起重装备全生命周期数据AI智能体平台
## 功能需求文档 v1.1

> **版本说明：** v1.1 在 v1.0 基础上，根据交互评审结果，对「设备全生命周期数据采集」模块进行了交互优化，重点解决上传入口不清晰、预处理效果不可感知、批量上传缺乏整体进度感知等问题。

## 一、项目概述

**项目名称：** 起重装备全生命周期数据AI智能体平台（EMDS-V2）

**项目目标：** 面向起重装备（起重机）使用单位，提供设备区域管理、设备全生命周期数据采集与管理、AI辅助数据处理（图像预处理、OCR识别、结构化）、数据结构化结果展示、隐患排查、风险评估、法规标准知识库等核心功能的综合性智能管理平台。

**技术栈：** Vue 3 + TypeScript + Pinia + Tailwind CSS + ECharts + Vite

### 模块五：设备全生命周期数据采集（独立功能）

> **说明：** 数据采集是独立的核心功能模块，与数据服务AI助手相互独立。用户在主应用页选择设备后，通过上方数据分类Tab选择数据类型，再选择日期上传对应的照片、扫描件或电子文档。文件上传后，系统自动在后台对图片/PDF进行图像预处理，处理结果可在采集页内直接预览。结构化的分析结果则在「数据服务AI助手」中查阅。

#### 5.1 操作流程

```
1. 左侧选择设备
       ↓
2. 顶部选择数据分类 Tab（如：点巡检数据、故障数据等）
       ↓
3. 左侧日历默认定位到今日，可切换至其他日期
       ↓
4. 右侧上传区：拖拽或点击选择文件（照片、PDF扫描件、电子文档）
       ↓
5. 系统自动触发图像预处理（去噪、矫正、增强对比度等），文件行实时显示处理状态
       ↓
6. 预处理完成后，点击文件行「查看效果」可在右侧抽屉对比处理前后效果
       ↓
7. 数据进入待结构化队列（结构化分析结果在「数据服务AI助手」中呈现）
```

#### 5.2 主内容区 Tab 导航总览

```
[点巡检数据] [故障数据] [维修数据]  |  [检验数据] [检测数据] [风险数据]  |
[运行数据*] [监控数据*] [监测数据*]  |  [基础信息] [出厂数据]
（*：待上线，当前禁用）
```

**关键交互规则：**

| 历史文件导入说明 | 「批量导入历史文件」按钮附带 tooltip：「使用文件原始修改日期归档，而非今日」，与普通上传明确区分 |

#### 5.4 文件列表行设计

**图片 / PDF 文件行（支持预处理）：**
```
┌────┬──────────────────────┬────────────┬──────────────────┬───────────┬────┐
│缩略图│ 文件名               │ 大小 · 日期 │ 预处理状态        │ [查看效果] │ 删 │

```

**其他格式文件行（不触发预处理）：**
```
┌────┬──────────────────────┬────────────┬──────────────┬────┐
│图标 │ 文件名               │ 大小 · 日期 │ Excel / PDF  │ 删 │
```

**预处理状态显示规则：**

| 状态 | 图标 | 说明 |
|------|------|------|
| 待处理 | 灰色圆圈 | 文件上传完成，预处理排队中 |

#### 5.5 预处理效果预览（图片/PDF 专属）

点击文件行「查看效果」按钮，在页面右侧以抽屉方式展开，**无需跳转至AI助手**：

```
┌──────────────────────────────────────────────┐
│  [← 关闭]  图像预处理预览  ·  spot_check.jpg  │
├─────────────────┬────────────────────────────┤
│  处理前（原图）   │  处理后（已优化）            │
│                 │                            │
│  [模糊/倾斜图]  │  [清晰/矫正图]  ✓ 已完成    │
│                 │                            │
│  倾斜 3.2°      │  文档质量评分               │
│  模糊 · 污渍    │  62 → 95                   │
├─────────────────┴────────────────────────────┤
│  处理项: [倾斜矫正 ✓] [清晰化 ✓] [污渍去除 ✓] │
└──────────────────────────────────────────────┘
```

- 抽屉从页面右侧滑出，不覆盖左侧日历和文件列表
- 预处理完成后自动高亮「查看效果」按钮，主动引导用户查看
- 三项处理操作（倾斜矫正 / 清晰化 / 污渍去除）支持单独开关后重新处理

#### 5.6 批量上传进度气泡

当单次上传文件数量 ≥ 2 个时，在页面右下角显示悬浮进度气泡：

```
┌─────────────────────────────┐
│  正在上传  3 / 5            │
│  ████████████░░░░  60%     │
│  预处理队列: 2 个等待中     │
│                    [收起 ∨] │
└─────────────────────────────┘
```

- 全部上传完成后气泡变为：`✓ 5个文件上传完成，预处理进行中`
- 3秒后自动消失
- 点击「收起」可折叠为角标小圆点，不遮挡主界面操作

#### 5.10 后台图像预处理流程

文件上传成功后，系统自动对图片和 PDF 扫描件触发预处理：

```
上传文件
  ├─ 图片（JPG/PNG）→ 去噪 → 倾斜矫正 → 对比度增强 → 输出标准化图像
  └─ PDF 扫描件    → 逐页渲染 → 同上图像处理 → 重新封装 PDF
```

- Excel / Word / CSV 等非图像文件不触发预处理，直接进入结构化队列
- 处理状态实时反馈到文件列表的「预处理状态」列
- 处理完成后「查看效果」按钮点亮，可在当前页直接查看前后对比
- 处理完成后数据自动进入待结构化队列，结构化结果在「数据服务AI助手」中呈现

### 模块六：数据服务AI助手

> **说明：** 数据服务AI助手是独立的数据结果查阅与分析服务，不承担文件上传功能。其核心职责是对已完成预处理的数据执行 OCR 识别与结构化，并将结构化结果以可视化、可检索的方式呈现给用户。

**入口：** 底部快捷栏「数据服务AI助手」按钮 或 菜单页对应卡片

**页面布局：**
```
┌───────────────────────────────────────────────────┐
│  [出厂数据] [点巡检数据] [检验数据] [检测数据]        │
│  [故障数据] [维修数据]                              │
├──────────────────────────────────────────────────┤
│  结构化数据结果列表（对应已处理的文件）               │
│  ┌──────────────────────────────────────────────┐│
│  │ 文件名 | 类型 | 上传时间 | OCR | 结构化 | 聚类 ││
│  │ xxx.pdf │ PDF │ 2024-01 │ ✓  │  ✓   │  ⏳  ││
│  └──────────────────────────────────────────────┘│
├──────────────────────────────────────────────────┤
│  处理结果面板（横向Tab切换）                         │
│  [OCR识别结果] [结构化数据] [聚类分析] [分析报告]     │
│  （对应面板展示结构化内容、图表、统计等）              │
└───────────────────────────────────────────────────┘
```

**结构化处理流程（4个阶段）：**

```
预处理完成的文件 → [OCR识别] → [数据结构化] → [聚类分析] → [分析报告]
```

| 阶段 | 面板名称 | 功能描述 |
|------|---------|---------|
| 1 | OCR识别结果 | 展示从图片/PDF中识别出的原始文字内容 |
| 2 | 结构化数据 | 将识别文字按字段提取为结构化表格（键值对、表单字段等） |
| 3 | 聚类分析 | 对多条结构化记录进行聚类分组，识别规律和异常 |
| 4 | 分析报告 | 生成可视化图表与文字分析结论 |

**处理状态标识：**

| 状态 | 显示样式 | 含义 |
|------|---------|------|
| 待处理(pending) | 灰色圆圈 | 尚未开始 |
| 处理中(processing) | 蓝色旋转动画 | 正在执行 |
| 已完成(completed) | 绿色 ✓ | 处理成功 |
| 出错(error) | 红色 ✕ | 处理失败，可重试 |

**AI处理进度抽屉（AIProcessDrawer）：**
- 右侧滑出抽屉面板
- 实时显示每个处理步骤进度与日志
- 支持查看每步骤结果详情

**检验/检测到期提醒：**
- 从结构化数据中自动解析检验/检测有效期
- 到期前推送醒目提醒，点击展开「提醒详情抽屉」（含到期日期、检验机构、联系方式等）
